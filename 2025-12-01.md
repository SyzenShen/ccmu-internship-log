这将近半年的实习，对我来说是一个把生物学想法变成可复用工具的过程。下面就从几个方面做一个小结。

---

## 一、实习背景与整体目标

从 2025 年 7 月到 11 月，我在首都医科大学相关实验室进行生物信息学方向的实习，并在 GitHub 上用 `ccmu-internship-log` 这个仓库，用按日期命名的 Markdown 文件记录每天的学习与反思。([GitHub][1])

我做的事情很简单：

1. 把常见的生物信息学分析流程“走通”一遍，并能独立复现；
2. 尽量把零散的脚本整理成**可复用、可维护的工具或系统**；
3. 用完整项目锻炼工程能力（后端、前端、部署、测试、维护）；
4. 通过持续写日志和文档，把这些经验沉淀下来。

---

## 二、生物信息学分析工作流

实习的第一部分，是围绕 `bioinfo-internship-project` 仓库，系统学习和整理生物信息学分析脚本与笔记。([GitHub][2])

### 1. R 语言绘图与数据处理

在这个过程中，我用 R 语言完成了多种数据可视化任务，包括：

* 对 7 万多行的大型文本数据做差值计算、异常值剔除，并绘制 **density plot（密度分布图）**，同时学习用 `summary()`、`str()` 等函数检查数据结构；([GitHub][2])
* 编写脚本完成 **GO 富集分析**，并通过条形图与 lollipop 图将结果可视化，着重练习如何在图上呈现富集条目、基因数以及显著性（如 −log10 p 值）；([GitHub][2])
* 绘制 **volcano plot + enrichment barplot** 的组合图，用于展示差异表达基因与功能富集结果，更接近论文级别图形。([GitHub][2])

通过这些练习，我熟悉了 `ggplot2` 等常用绘图方式，且逐渐习惯了**想清楚要表达什么生物学信息，再决定用什么图、什么尺度来呈现**。

### 2. 富集与通路可视化

在富集分析这一块，我进一步尝试了多种可视化方式，例如：([GitHub][2])

* 用 barplot 展示前 20 个富集通路（GO 或 KEGG），条形长度对应基因数，颜色代表显著性；
* 用 **Sankey + bubble 混合图** 同时展示通路层级关系和基因数量，使结果更直观；
* 编写脚本，将通路与基因以同步方式展示，方便从“通路—基因—生物学意义”这一链条去理解数据。

通过这些脚本的训练**让结果变得可讲述**。

### 3. Linux、流程梳理与自动化脚本

在分析侧，除了 R 可视化之外，我还围绕测序数据做了不少基础梳理和脚本练习：([GitHub][2])

* 写下了 `Quick_basic_tutorial_of_Linux.md`，整理常用 Linux 命令和在服务器上的基本操作；
* 通过 `NGS_preprocessing_to_variant_calling.md` 梳理了从 NGS 预处理到变异检测的一整套流程结构；
* 借助 Python 脚本对大型 Excel 表格进行自动填充、去重、补全空值等操作，把原本需要人工反复点选的工作，变成一键运行的自动化处理。([GitHub][2])

同时，我也按照自己的描述，使用本地 IGV 等软件对基因进行可视化，做裁切、拼接和对比，帮助把“脚本算出来的结果”和“真实基因结构”对应起来。

这一阶段的收获，是把**“会用工具”升级为“理解流程 + 能写脚本 + 知道在哪里检查结果”**。

## 三、MetaServe

实习的第二个重点，是在 `Download_system_project` 仓库中从零搭建并迭代一个完整的系统——MetaServe。([GitHub][3])

### 1. 系统定位与目标

MetaServe是一个基于 **Django + Vue** 的系统，主要面向科研机构或生物信息中心，用来替代传统的 FTP / NAS 文件系统：([GitHub][3])

* 管理、检索、追踪高通量测序数据及其派生文件；
* 通过文件索引、元数据检索、权限管理和可靠传输，为样本与文件的全生命周期提供统一入口；
* 与 Cellxgene 集成，实现 `.h5ad` 单细胞数据的即时可视化。([GitHub][3])

**该系统目前已在首都医科大学部署**，我在日常也会持续维护和优化。([GitHub][3])

### 2. 架构与关键模块

在这个项目中，我在实践中熟悉了一个完整 Web 项目的主要环节，包括：

* 后端：基于 Django 搭建 API，处理文件索引、元数据查询、权限控制等逻辑；
* 前端：基于 Vue 搭建界面，覆盖文件列表、检索界面、上传下载交互等；
* 可视化：通过与 Cellxgene 的集成，让 `.h5ad` 文件可以在 Web 端快速浏览，而不需要每次都在本地配置环境；([GitHub][3])
* 扩展接口：在 README 中保留了 `ml_interface` 作为 Intelligent Analytics Interface，用于未来接入自动标注、质控检测、摘要生成、向量检索以及 `.h5ad` 预处理等 AI / ML 模块。([GitHub][3])

对我来说，这个设计本身就是“可复用工具思维”的一次实践：系统一开始就为未来的算法扩展留好了位置。

### 3. 上线与维护

和以往写完脚本就结束不同，这次项目我经历了一个系统从开发到部署再到维护的过程：([GitHub][3])

* 在首都医科大学完成部署，并结合真实数据场景进行测试；
* 通过日志和简单测试脚本，定位并修复上传 / 下载等流程中的问题；
* 保持仓库结构清晰，配合文档，让后续维护和功能扩展变得更可控。

MetaServe对我来说，不只是一个课程作业，而是一个真正“被用起来”的系统，也让我意识到：**工程化能力本身就是科研生产力的一部分**。

## 六、综合收获与下一步计划

综合来看，这将近半年的实习，我主要有几方面收获：

1. **生物信息学实践能力**

   * 能够基于 R 完成从数据清洗到可视化的完整流程（包括密度图、火山图、富集条形图、Sankey 混合图等）；([GitHub][2])
   * 熟悉利用 Linux 环境、脚本和本地可视化工具（如 IGV）处理和检查测序数据结果。

2. **工程化与系统思维**

   * 完成并上线了一个围绕高通量测序数据管理的 Django + Vue 系统（MetaServe / BioFileManager），并在首都医科大学实际部署使用；([GitHub][3])
   * 从一开始就考虑“未来如何扩展”和“如何让别人复用”，在系统里预留了智能分析接口。

3. **工具化与好奇心驱动的学习方式**

   * 通过 Utility-Box 等小项目，把日常遇到的数字问题整理成可复用的小工具和公开教程；([GitHub][4])
   * 逐渐形成了一个习惯：**凡是会重复的操作，就尽量写成脚本或文档。**

4. **记录与反思能力**

   * 通过连续几个月的 GitHub 日志，把实习中的学习轨迹完整保存下来，让“我当时为什么这么做”这类问题，以后可以有迹可循。([GitHub][1])
